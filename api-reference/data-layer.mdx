## `GET /api/v1/data-layer`

Returns the local data layer specification, cloud layer configuration, and data ownership rules for the Mukoko ecosystem.

The architecture has **three sources of truth**: Supabase (relational), ScyllaDB (non-relational), and the Web3 Pod (personal/sovereign). Data flows across **seven layers**: Pod, Relational (Supabase), Document (ScyllaDB), Sync (CouchDB), Edge (Cloudflare KV + Durable Objects), Device (RxDB/SQLite), and Open Data (Doris). CouchDB is the **sync protocol**, not a data store — it replicates data between layers. ScyllaDB is the non-relational source of truth, running on Fly.io in Johannesburg. Doris serves double duty as both the analytics engine and the live search engine.

### Response Shape

| Field | Type | Description |
|---|---|---|
| `localLayer` | `object` | Local-first storage strategy (RxDB, SQLite, IndexedDB) |
| `cloudLayer` | `object` | Cloud sync targets (CouchDB, Supabase) |
| `dataOwnership` | `object` | Rules governing user data ownership and portability |

### Example Local Layer

```json
{
  "engine": "RxDB",
  "backends": [
    { "name": "SQLite", "platform": "native", "useCase": "Capacitor mobile/desktop apps" },
    { "name": "IndexedDB", "platform": "browser", "useCase": "PWA and web apps" }
  ],
  "features": ["reactive-queries", "schema-validation", "replication"]
}
```

### Example Cloud Layer

```json
{
  "sync": [
    { "name": "Supabase", "purpose": "Relational source of truth — auth, wallet, transactions" },
    { "name": "ScyllaDB", "purpose": "Non-relational source of truth — high-throughput document data" },
    { "name": "Apache CouchDB", "purpose": "Sync protocol — replicates data between layers" }
  ]
}
```

### Example Data Ownership Rule

```json
{
  "rule": "export-anywhere",
  "description": "Users can export all their data in open formats at any time."
}
```
